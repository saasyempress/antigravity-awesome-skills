# 50_publish_packets

Purpose:
This folder holds finalized, packaged content ready for publishing.

What enters this folder:
- Human approved content from 40_approved
- Associated metadata and references

What lives here:
- Final copy
- Metadata for platform, format, and timing
- Source references and attribution
- Approval records

Rules:
- No drafting happens here
- No ideation happens here
- Content here is considered final

Publishing:
- Content from this folder may be published immediately or scheduled
- Publishing cadence is controlled by the human, not the system
